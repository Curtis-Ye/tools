# .gitignore学习
## 什么是.gitignore?
.gitignore是一个文本文件，放在Git仓库的根目录下（也可以是子目录，但通常建议放在根目录），其中的每一行都是一条"忽略规则"。Git在检测文件变化时，会跳过.gitignore中匹配到的文件/目录，不将它们加入暂存区（git add），也不会在提交历史中出现。
## 为什么需要.gitignore?
1.避免提交临时文件（如编译产物、缓存文件）  
2.排除敏感信息（如配置文件中的密码、密钥）  
3.忽略依赖包（如node_modules、venv），这些可以通过包管理工具重新安装  
4.过滤IDE/编辑器的配置文件（如.idea、.vscode），避免团队成员因工具不同产生冲突  
## .gitignore的基础语法
主要用法是编辑.gitignore文件。  
1.直接写文件名：匹配同名文件（如debug.log）  
2.通配符*：匹配任意个任意字符（如*.log匹配所有.log结尾的文件）  
3.通配符?：匹配单个任意字符（如file?.txt匹配file1.txt、fileA.txt，但不匹配file12.txt）  
4.通配符**：匹配多级目录（如** /dist匹配所有目录下的dist文件夹，包括根目录的dist和src/dist）  
## 目录匹配：/ 的用法
结尾加/：表示匹配目录（而非文件）。例如node_modules/只忽略node_modules目录，而node_modules可能会匹配同名文件  
开头加/：表示只匹配仓库根目录下的文件/目录。例如/tmp只忽略根目录的tmp，不忽略src/tmp  
## 否定规则：! 开头的行
如果某个文件被前面的规则忽略了，但你想"例外"地保留它，可以用!否定。  
例如：  
```
*.log       # 忽略所有.log文件
!important.log  # 但不忽略important.log
```
**注意：否定规则只对前面的规则生效，且如果父目录被忽略，子文件无法被否定。**
例如：  
```
build/      # 忽略build目录
!build/index.html  # 无效！因为build/已被忽略，子文件无法单独保留
```
